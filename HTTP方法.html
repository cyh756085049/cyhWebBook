HTTP方法

HTTP/1.1 方法

GET ：获取资源。

POST：传输实体主体 。

PUT：传输文件 。

HEAD：获得报文首部。和 GET 方法一样，只是不返回报文主体部分。

DELETE：删除文件 。是与 PUT 相反的方法。DELETE 方法按请求 URI 删除指定的资源。

OPTIONS：用来查询针对请求 URI 指定的资源支持的方法。

TRACE：追踪路径 。让 Web 服务器端将之前的请求通信环回给客户端的方法。客户端通过 TRACE 方法可以查询发送出去的请求是怎样被加工修改/篡改的。

CONNECT：要求用隧道协议连接代理 。在与代理服务器通信时建立隧道，实现用隧道协议进行 TCP 通信。
